Задание № 6 (последнее перед курсовым проектом) Веб-приложение (используйте стек вашей группы). Реализуйте «программное обеспечение для совместной презентации» для всех. 
Никакой регистрации или аутентификации, все пользователи имеют немедленный доступ к списку «презентаций» (пользователь только вводит произвольный ник для входа). 
Каждый пользователь может создать презентацию или присоединиться к существующей презентации. Создатель презентации может видеть список подключенных пользователей (скажем, справа) 
и может сделать любых других пользователей «редакторами» (или переключить их обратно на «зрителей»). Зритель может только просматривать совместную работу (никаких инструментов 
редактирования). Несколько редакторов могут редактировать одни и те же или разные слайды (только создатель может добавлять новые слайды или удалять их). Когда кто-то что-то 
редактирует, это отображается для других пользователей «сразу» (может быть небольшая задержка, вы можете либо опросить сервер, либо, что предпочтительнее, использовать веб-сокеты). 
Все отредактированное на слайдах сохраняется «навсегда» (если пользователь присоединится к этой презентации позже, он/она увидит все, что было создано до этого). Область слайда 
должна заполнять все окно (кроме верхней панели инструментов, левой панели слайдов и правой панели пользователей) и масштабироваться/прокручиваться адекватно. По крайней мере, 
добавьте возможность добавлять редактируемые, перемещаемые текстовые блоки с форматированием markdown. Конечно, в презентационном программном обеспечении должен быть режим 
«настоящего».
Постарайтесь сделать его точным, последовательным, похожим на готовое к бизнесу приложение. Я знаю, что найдется один или два человека, которые попытаются отправить что-то 
уродливое, например: «Но это работает — посмотрите, вот страница с кнопкой!» Пожалуйста, не надо. Сделайте так, чтобы это стоило потраченного времени.
Необязательное требование (каждое из них повысит оценку):

* Возможность стирать ранее нарисованные элементы.,
* Несколько дополнительных фигур («прямоугольник», «круг», «стрелка» и т. д.) с цветами.,
* Изображения.,
* Функциональность увеличения/уменьшения масштаба.,
* Предварительный просмотр миниатюры первого слайда в списке презентаций.,
* Анимации и переходы.,
* Возможность экспорта в PDF.,

Обратите внимание, что вам не нужно писать низкоуровневый код для отслеживания позиций событий мыши на холсте или что-то в этом роде. ИСПОЛЬЗУЙТЕ БИБЛИОТЕКИ ДЛЯ ВСЕГО.

Task #6 (the last before the course project) Web-application (use the stack of your group). Implement a "collaborative presentation software" for everyone. No registration or 
authentication, all users have immediate access to the "presentations" list (the user only provides arbitrary nick-name to enter). Each user can create presentation or join 
existing presentation. Presentation creator can see the list of the connected users (let's say on the right) and can make any other users "Editors" (or switch them back to 
"Viewers"). Viewer can only view the collaborative work (no editing tools). Several editors can edit the same or different slides (only creator can add new slides or remove them).
 When somebody edit something, it appears to other users "immediately" (there may be a slight delay, you can either poll the server or preferably use websockets). Everything 
 edited on the slides is stored "forever" (if user joins this presentation later, he/she sees everything what was created before). Slide area should fill the whole window (except 
 the top tool panel, left slides panel and right users panel) and scale/scroll adequately. At least, add possiblity to add editable, movable text blocks with markdown formatting. 
 Of course, in the presentation software there should be "present" mode.
Try to make it accurate, consistent, looking like a business-ready application. I know there will be one or two people who try to submit something ugly like, 'But it works—look, 
there’s a page with a button!' Please don’t. Make it worth the time spent.
Optional requirement (each will increase the grade): 

* Ability to erase previously draw elements.,
* Multiple additional shapes ("rectangle", "circle", "arrow" etc.) with colors.,
* Images.,
* Zoom in/out functionality.,
* Preview thumbnail image of the first slide in the presentation list.,
* Animations and transitions.,
* Export to PDF option.,

Please, note that you don't need to write low level code for mouse event tracking positions on a canvas or something like this. USE LIBRARIES FOR EVERYTHING.
